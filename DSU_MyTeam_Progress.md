## 📋 1. 프로젝트 기본 기조 및 아키텍처 정보
*   **프로젝트명**: DSU My_team (동서대학교 MYDEX 팀 매칭 시스템)
*   **개발 기조**: 하나의 `web/index.html` 파일 내부에서 자바스크립트를 이용해 페이지 새로고침 없이 즉각 화면을 전환하는 **단일 페이지 애플리케이션(SPA)** 방식으로 웹 구축.
*   **디자인 규격**: TailwindCSS 기반의 **Pastel Ethereal 테마**, 폰트는 **Plus Jakarta Sans** 사용.
*   **데이터 연동 구조**: 웹 화면(Vanilla JS) ↔ 내장 API 서버(aiohttp, Port 8080) ↔ 로컬 DB(SQLite) + 클라우드 DB(Google Sheets).

### 📅 2026-06-02 (Claude Sonnet 4.5 - 버그 수정 세션)
*   **[🔴 치명적 버그 수정 - 무한 로딩 해결]**
    *   **버그 원인**: `loadDashboardPrograms()` 함수가 이전 Gemini 작업에서 `navigate()` 함수 내부에 중첩 선언되어 있어, `DOMContentLoaded` 이벤트에서 호출 시 전역 스코프에서 접근 불가능한 상태였음. 함수 자체가 실행되지 않아 HTML의 하드코딩된 "Loading..." 템플릿만 무한 표출되는 현상 발생.
    *   **조치 1**: `category-hot`, `category-design`, `category-ai` 3개 컨테이너 내부의 하드코딩 로딩 템플릿(12개 카드) 전부 삭제 → 빈 컨테이너로 교체.
    *   **조치 2**: `loadDashboardPrograms()` 함수를 `navigate()` 밖 전역 스코프로 완전히 이동 재선언.
    *   **조치 3**: 렌더링 방식 전면 교체 - DOM 텍스트 수정 방식 폐기 → `innerHTML = ''` 후 `insertAdjacentHTML`로 카드 HTML 전체를 동적 생성하는 방식으로 재작성.
    *   **조치 4**: `onerror` 속성을 카드 img 태그에 추가해 이미지 로드 실패 시 아이콘만 표출되도록 안전 처리.
    *   **조치 5**: 4종류 고해상도 Unsplash 더미 이미지를 순환 배치하여 썸네일 없는 카드에 자동 삽입.
    *   **조치 6**: `fetch` 성공/실패 시 `console.log`/`console.error`로 디버깅 로그 추가, 실패 시 빨간 에러 메시지 UI 즉시 표출.
